Пример иерархической структуры работ (ИСР) для IT-проекта:

1. Анализ требований
   - Комментарий: Собрать требования от заказчика, составить ТЗ
   1.1. Интервью с заказчиком
        - Комментарий: Провести встречи с ключевыми пользователями
   1.2. Анализ существующих процессов
        - Комментарий: Изучить текущую документацию

2. Проектирование
   - Комментарий: Создать архитектуру решения
   2.1. Проектирование БД
        - Комментарий: Определить схему данных
   2.2. Проектирование API
        - Комментарий: Спецификация эндпоинтов

3. Разработка
   - Комментарий: Реализация функционала
   3.1. Бэкенд разработка
   3.2. Фронтенд разработка

4. Тестирование
   - Комментарий: Проверить соответствие требованиям
   4.1. Модульное тестирование
   4.2. Интеграционное тестирование
   4.3. Приемочное тестирование

5. Внедрение
   - Комментарий: Развертывание и обучение
   5.1. Подготовка окружения
   5.2. Развертывание
   5.3. Обучение пользователей
